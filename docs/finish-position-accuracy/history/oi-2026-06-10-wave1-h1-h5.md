## What was tried

This is the **Wave 1** relentless-improvement push for tomorrow's 2026-06-10 大井 (Ōi, `keibajo_code=44`) NAR card, following the iter34 R5 similarity-member rejection ([`oi-2026-06-10-r5-nar-similarity-member.md`](oi-2026-06-10-r5-nar-similarity-member.md)). R5 closed by demanding a **genuinely-new signal that is BOTH orthogonal AND finish-predictive**. Wave 1 attacks that target on **five fronts in parallel**, each a distinct hypothesis about where unextracted finish signal might live, all judged on the **powered full-NAR per-class holdout** (n = 26060 C / 7217 other / 7124 B / 2812 A …) plus an Ōi(44) cross-check, under the strengthened gate (nested split + multi-metric ≥2/4 axes + ≥1 place positive + no axis < −0.05pp + top1 paired-bootstrap LB95>0 + Holm).

The five Wave 1 hypotheses:

- **H1 — field-relative / competitive-context features.** Within-race rank/zscore transforms of the existing absolute-level signals (market, speed, weight) as a competition-aware channel.
- **H2 — momentum / trajectory / freshness features.** Short-minus-long form deltas, recency slopes, freshness, weight change, consecutive-improvement counts — orthogonal-in-time signal distinct from absolute level.
- **H3 — calibration + meta-learner stacking.** Ridge / logistic / LGBM stack combiners and isotonic calibration over the existing per-class members.
- **H4 — HPO + alternative loss.** Per-class CatBoost hyperparameter search with alternative ranking losses. **Still running at the time of this record** — verdict to be appended.
- **H5 — multi-class probability head.** A single 7-class prob-head over all NAR classes. **Cut on memory** before producing a verdict (see below).

## Per-bucket findings

The decisive pattern across H1/H2/H3 is identical and reinforces iter34 R5: a hypothesized "new" channel turns out to be **already present** in the 174-feature baseline (field-relative ranks, recency/days-since/form-trend/weight-diff, within-race-normalized market), so the new features land at ~0 importance and the member is ~0.999-correlated with the existing residual. Where a transform is genuinely **orthogonal** (odds ordinal-rank ρ~0.17; the R5 similarity geometry), its finish signal is **sub-noise**. The recurring **near-miss** is a class that looks directionally good on the holdout (H1 `other` 4/4 axes positive; H2 `C` top1 +0.161pp; H3 `B` top1 +0.253pp) but whose top1 bootstrap **LB95 lands just <0** or whose **place axis breaches −0.05pp** — exactly the shape the strengthened gate exists to reject.

### Root-cause finding (the headline)

The **174-feature baseline is comprehensive**: it ALREADY contains every channel Wave 1 proposed — field-relative rank/diff cols, recency/days-since/form-trend/weight-diff cols, and within-race-normalized market signal. New features in those channels are **redundant**. The genuinely-orthogonal transforms (odds ordinal-rank, similarity geometry) carry **sub-noise** finish signal. Combined with the NAR/Ōi holdout sizes, the available gains (**~0.05–0.16pp**) are **below the bootstrap-confirmable threshold**, so the strengthened gate **correctly rejects** them. **This is the accuracy CEILING for same-night feature/ensemble levers** over the current pipeline.

## Next iteration recommendation — path forward

Stop pulling **same-night feature/ensemble levers** — Wave 1 (H1/H2/H3, H4 pending) closes the same-night search: the feature set is **comprehensive and saturated**. Real gains require **genuinely-NEW DATA not in the pipeline**, all of which are **multi-day data-acquisition** efforts (not same-night levers):

1. **Odds/market MOVEMENT over time** — the drift/velocity of odds before the off, not just the final odds the pipeline already normalizes.
2. **Sectional / pace-scenario modeling** — lap-by-lap pace dynamics and the resulting race shape.
3. **Track-bias-on-the-day** — inside/outside and front/closer bias measured per meeting.
4. **Weather / going dynamics** — evolving track condition over a card.

Each must clear the same bar Wave 1's levers could not: **incremental in target space**, validated under the nested-split + multi-metric + paired-bootstrap-LB + Holm gate on the powered full-NAR holdout, trained via a multi-hour full retrain or HPO/L4 — not a blend, feature bolt-on, or similarity member over the existing signal.
